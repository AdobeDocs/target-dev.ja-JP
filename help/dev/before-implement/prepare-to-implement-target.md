# [!DNL Target]の実装の準備

[!DNL Adobe Target]の実装を準備する際に完了するタスクと収集する情報。

これらのガイドラインに従い、提供された質問に記入すると、自分で実装するか、[!DNL Adobe Consulting]で作業するか（推奨）に関わらず、[!DNL Target]を実装するのに役立ちます。

Adobe コンサルタントと連携する場合は、[!DNL Target]を実装する前に、これらのドキュメントに記載されている情報を知っている必要はありません。 ただし、コンサルティングが作成するドキュメントと彼らが尋ねる質問に熟知していることは、実装前のプロセスの迅速化に役立ちます。 これらのドキュメントは、将来変更される可能性があります。

## 情報の[!DNL Adobe Target]件のビジネスリクエスト

このセクションには、[!DNL Target]実装キックオフ呼び出しの前にお客様が完了する必要があるビジネス情報要求（RFI）文書が含まれています。

次にビジネス RFI ドキュメントの目的を示します。

* クライアントのビジネス目標の把握
* クライアントの最適化目標の把握

[ビジネス情報提供依頼書](assets/business-rfi.docx)をダウンロードしてください。

## [!DNL Adobe Target]件の技術的な情報リクエスト

このセクションには、コンサルティングチームがクライアントのビジネス要件と技術要件を満たす[!DNL Target]実装を開発するのに役立つ技術情報要求（RFI）ドキュメントが含まれています。 このドキュメントの質問は、実装を見直す必要性を最小化または除外するために、すべての隠れたハードルや障害を明らかにします。

次に技術 RFI ドキュメントの目的を示します。

* 技術プラットフォームおよびあらゆる制限の把握
* 人およびプロセスとの関連で、技術環境の把握

[技術情報提供依頼書](assets/technical-rfi.docx)をダウンロードしてください。
